# webar-project

A-FrameとAR.jsを利用した、専用アプリ不要でスマホのWebブラウザから手軽に体験できるARプロジェクトです。

## 🔗 URLの仕組みとアクセス方法

ベースURLの末尾に `?id=【モデル名】` のパラメータをつけることで、指定したモデルを動的に呼び出します。

* **アクセスURL（例）**: `https://bpm82.github.io/webar-project/?id=【モデル名】`
*※【モデル名】は、後述の `assets` 内のフォルダ名と完全に連動しています。

## 📁 ディレクトリ・ファイル構成と追加ルール

新しいモデルを追加する際は、`assets` フォルダ内に新しい【モデル名】のフォルダを作成し、以下の **3点セット** を必ず同じ名前で格納してください。

1. `model.glb` （3Dモデル本体）
2. `pattern.patt` （ARマーカーファイル）
3. `qr.png` など （読み込み用QRコード画像）

### ⚠️ GitHubでのフォルダ作成に関する注意
GitHubの仕様上、中身が空のフォルダは作成・保存できません。Web画面から新規フォルダを作成する場合は、`assets/【モデル名】/.keep` などのダミーファイル名を入力して一度フォルダ枠を確保してから、実データをアップロードしてください。

## 🛠️ マーカーの作成 (AR.js Marker Training)

ARマーカー（.patt ファイル）の生成は以下の公式ジェネレーターを使用します。
https://jeromeetienne.github.io/AR.js/three.js/examples/marker-training/examples/generator.html

## 🧊 Blenderからのモデル書き出し（エクスポート）ルール

ARで表示されない、またはファイルサイズが異常に小さい（例：176バイトで空っぽになる）場合は、Blenderの書き出し設定で以下を徹底してください。

1.書き出す対象のオブジェクトをクリックして「選択（オレンジ色のフチドリ）状態」にする。
2.エクスポート設定で**「モディファイアーを適用」**に必ずチェックを入れる（厚みや丸みをつけている場合）。
3.フォーマットは `glTF バイナリ (.glb)` でエクスポートする。
4.書き出し後、PC上でファイルサイズが数KB〜数MBあるか確認してからGitHubへアップロードする。

## ⚠️ よくあるトラブルと仕様

* **モデルが寝てしまう場合**: Blender（Z軸が上）とWebAR（Y軸が上）で座標系のルールが異なるためです。現状は `rotation="0 0 0"` で設定していますが、もし倒れる場合はHTML内のプログラムで `rotation="-90 0 0"` などに設定して強制的に起こしてください。スマホで反映されない場合は、キャッシュが悪さをしているため、URLの末尾に `&v=2` 等をつけてアクセスし直してください。
* **「動作と方向へのアクセスを求めています」の許可 (iOS)**: スマホの傾き（ジャイロセンサー）を検知し、空間の立体感や安定性を保つための必須機能です。ワークショップ時は、お客さんに必ず「許可」を押してもらうよう案内してください。
* **個人情報の保護**: ワークショップ等でお客さんが作ったモデルをアップロードする際、フォルダ名（モデル名）やテクスチャに本名（フルネーム等）は使用しないでください（連番やニックネーム推奨）。
