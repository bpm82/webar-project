# webar-project

A-FrameとAR.jsを利用した、専用アプリ不要でスマホのWebブラウザから手軽に体験できるARプロジェクトです。

## 🔗 URLの仕組みとアクセス方法

ベースURLの末尾に `?id=【モデル名】` のパラメータをつけることで、指定したモデルを動的に呼び出します。

* **アクセスURL（例）**: `https://bpm82.github.io/webar-project/?id=【モデル名】`
* ※【モデル名】は、後述の `assets` 内のフォルダ名と完全に連動しています。

## 📁 ディレクトリ・ファイル構成と追加ルール

新しいモデルを追加する際は、`assets` フォルダ内に新しい【モデル名】のフォルダを作成し、以下の **3点セット** を格納してください。

1.  `model.glb` （3Dモデル本体） **※プログラムが探すので、絶対にこの名前にする！**
2.  `pattern.patt` （ARマーカーファイル） **※プログラムが探すので、絶対にこの名前にする！**
3.  `qr.png` など （読み込み用QRコード画像） **※この画像の名前は自由でOKです！**

### ⚠️ ファイルのアップロード手順（迷子防止！）
初心者が一番失敗しやすいポイントです。GitHubのトップ画面にそのままファイルをドラッグ＆ドロップすると、プログラムが読み込めずエラーになります。必ず以下の手順でアップロードしてください。

1. GitHubの画面で `assets` フォルダをクリックし、さらに自分で作った `【モデル名】` のフォルダをクリックして中に入る。
2. 画面右上の **「Add file」** をクリックし、**「Upload files」** を選ぶ。
3. 用意した3点セット（.glb, .patt, .png）を画面にドラッグ＆ドロップする。
4. 緑色の「Commit changes」ボタンを押して保存する。

### ⚠️ GitHubでのフォルダ作成に関する注意
* GitHubの仕様上、**中身が空っぽのフォルダは作れません**。
* 画面上から新しくフォルダを作りたい時は、ファイル名の入力欄に `assets/【モデル名】/test` のように打ち込んでください。一番最後の `test` が「フォルダの場所取りをするための適当な仮ファイル」になります。これで一度保存（Commit）してフォルダを作ってから、実データをアップロードしてください（testファイルは放置でOKです）。
* **個人情報の保護**: ワークショップ等でお客さんが作ったモデルをアップロードする際、フォルダ名（モデル名）やテクスチャに本名（フルネーム等）は使用しないでください（連番やニックネーム推奨）。

## 🛠️ QRコードとマーカーの作成手順

1. **QRコードの作成**: 
   QRコード作成サイト（ https://qr.quel.jp/ ）などにアクセスURL（例: `https://bpm82.github.io/webar-project/?id=flower`）を入力し、QRコード画像を作成して保存します。
2. **ARマーカーの作成**: 
   以下の公式ジェネレーターにアクセスし、中央の「UPLOAD」から先ほどのQRコード画像をアップロードします。
   https://jeromeetienne.github.io/AR.js/three.js/examples/marker-training/examples/generator.html
   * 黒枠の中にQRコードが合体したら、**「DOWNLOAD MARKER」**を押して `.patt` ファイルを保存します。
   * 続けて、**「DOWNLOAD IMAGE」**も必ず押し、印刷や画面表示に使うためのマーカー画像（`.png`）をダウンロードして保存してください。

## 🧊 Blenderからのモデル書き出し（エクスポート）ルール

ARで表示されない、またはファイルサイズが異常に小さい（例：176バイトで空っぽになる）場合は、Blenderの書き出し設定で以下を徹底してください。

1.  書き出す対象のオブジェクトをクリックして「選択（オレンジ色のフチドリ）状態」にする。
2.  エクスポート画面の右側メニューにある**「データ（またはメッシュ）」**の項目を開き、**「モディファイアーを適用」**に必ずチェックを入れる（グラスの厚みや丸みをつけている場合）。
3.  フォーマットは `glTF バイナリ (.glb)` でエクスポートする。
4.  書き出し後、PC上でファイルサイズが数KB〜数MBあるか確認してからGitHubへアップロードする。

## ⚠️ よくあるトラブルと仕様

* **モデルが寝てしまう場合**: Blender（Z軸が上）とWebAR（Y軸が上）で座標系のルールが異なるためです。現状はHTML内のプログラムで `rotation="-90 0 0"` などに設定して強制的に起こしています。スマホで反映されない場合は、キャッシュが悪さをしているため、URLの末尾に `&v=2` 等をつけてアクセスし直してください。
* **「動作と方向へのアクセスを求めています」の許可 (iOS)**: スマホの傾き（ジャイロセンサー）を検知し、空間の立体感や安定性を保つための必須機能です。ワークショップ時は、お客さんに必ず「許可」を押してもらうよう案内してください。
